Overview

​Enza REST API Documentation

Welcome to the Enza REST API Developer Portal. Our API provides seamless integration for instant issuing services across Africa. This portal offers all the necessary tools, detailed documentation, and API endpoints required to empower your financial operations with real-time card issuance capabilities. Explore the portal, integrate with ease, and unlock secure, efficient, and scalable instant issuing solutions.

Endpoint:https://uat-api.enza.cloud/TX/v1

​Get Card Info

GET
https://uat-api.enza.cloud/TX/v1
/cards/by-rid/{rid}

Retrieve detailed information of a card by its RID (External ID).

​path Parameters

  • rid
    string
    required

    The unique identifier for the card (External RID).

​Responses

    • card
      object
      optional
GET /cards/by-rid/{rid}

​Change Ecom Status

POST
https://uat-api.enza.cloud/TX/v1
/cards/tdsenrollement

Modify the Ecom Status

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardRefNum
    string

    The PAN of the card whose Ecom Status is being updated.

  • ECstatusEnable
    integer

    Ecom Status (e.g., 1 for enrolled, 0 for not enrolled).

​Responses

    • message
      string

      Card Status changed successfully! (Active)

POST /cards/tdsenrollement

​Activate Card

POST
https://uat-api.enza.cloud/TX/v1
/cards/activate

Activate a card by providing the CardStatus and CardRefNum.

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardStatus
    string

    The status of the card.

  • CardRefNum
    string

    The CardRefNum of the card to be activated.

​Responses

    • message
      string

      Card Activated successfully!

POST /cards/activate

​DeleteToken Request

POST
https://uat-api.enza.cloud/TX/v1
/cards/deletetoken

Remove a token associated with a card using the CardRefNum.

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardRefNum
    string

    The reference number of the card whose token is being deleted.

​Responses

    • message
      string

      Confirmation message that the token is deleted.

POST /cards/deletetoken

​Retrieve Card Transactions

POST
https://uat-api.enza.cloud/TX/v1
/cards/retrievetransactions

Retrieve a list of transactions associated with a specific card.

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardRefNum
    string

    The reference number of the card whose transactions are being retrieved.

​Responses

    • Kind
      string

      The type of response, indicating transactions.

    • HasMore
      boolean

      Indicates if there are more transactions available beyond the current response.

POST /cards/retrievetransactions
